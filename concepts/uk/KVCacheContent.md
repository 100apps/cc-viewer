# Вміст KV-Cache

## Що таке KV-Cache?

KV-Cache (Key-Value Cache) — це основний механізм прискорення під час виведення у великих мовних моделях. Коли ви спілкуєтесь з Claude, кожен запит надсилає повну історію розмови (system prompt + визначення інструментів + історичні повідомлення). Якби все потрібно було обчислювати заново кожного разу, це було б дуже затратно за часом і коштами.

Роль KV-Cache полягає в: збереженні вже обчисленого вмісту в кеші, щоб його можна було повторно використовувати при наступному запиті, пропускаючи повторні обчислення.

## Як працює кеш

Prompt caching від Anthropic об'єднує ключі кешу у фіксованому порядку:

```
System Prompt → Tools → Messages (до cache breakpoint)
```

Поки цей префікс ідентичний попередньому запиту, API потрапить у кеш (повертає `cache_read_input_tokens`), що різко зменшує затримку та витрати.

## Що таке "поточний вміст KV-Cache"?

"Поточний вміст KV-Cache", відображений у cc-viewer, — це вміст, витягнутий з останнього запиту MainAgent і позначений як кешований. Конкретно включає:

- **System Prompt**: Системні інструкції Claude Code, включаючи ваш CLAUDE.md, опис проекту, інформацію про середовище тощо
- **Messages**: Кешована частина історії розмови (зазвичай більш ранні повідомлення)
- **Tools**: Список доступних визначень інструментів (таких як Read, Write, Bash, інструменти MCP тощо)

Цей вміст позначається в тілі запиту API через `cache_control: { type: "ephemeral" }` для позначення межі кешу.

## Чому варто переглядати вміст кешу?

1. **Розуміння контексту**: Знайте, який вміст Claude наразі "пам'ятає", і допоможіть собі оцінити, чи поведінка відповідає очікуванням
2. **Оптимізація витрат**: Влучення в кеш коштує набагато менше, ніж перерахунок. Перегляд вмісту кешу допомагає зрозуміти, чому певні запити спричинили перебудову кешу (cache rebuild)
3. **Налагодження розмови**: Коли відповідь Claude не відповідає очікуванням, ви можете перевірити вміст кешу, щоб підтвердити, що system prompt і історичні повідомлення правильні
4. **Навігація за інструкціями користувача**: Через список повідомлень користувача у вмісті кешу ви можете швидко перейти до будь-якої позиції в розмові
5. **Моніторинг якості контексту**: Під час щоденного налагодження, зміни конфігурації Claude Code або коригування промптів, KV-Cache-Text забезпечує централізовану перспективу, яка допомагає вам швидко підтвердити, що основний контекст не погіршився або не був забруднений неочікуваним вмістом — без необхідності переглядати оригінальні повідомлення одне за одним

## Життєвий цикл кешу

- **Створення**: При першому запиті або після втрати чинності кешу, API створює новий кеш (`cache_creation_input_tokens`)
- **Влучення**: Коли наступні запити мають узгоджений префікс, кеш повторно використовується (`cache_read_input_tokens`)
- **Закінчення**: Кеш має TTL (час життя) 5 хвилин і автоматично закінчується після цього часу
- **Перебудова**: Коли system prompt, список інструментів, модель або повідомлення змінюються, ключ кешу не збігається, що спричиняє перебудову
